Session 07: Error Handling & Resilience

⏱ 25 min · 🌱 Beginner · 📝 5 Quiz Questions

Learn

HTTP Error Codes

  • 401 — Bad or missing API key. Check your Authorization header.
  • 402 — Insufficient credits. Use :free models if you're out of credits.
  • 429 — Rate limited. You're sending requests too fast — back off and retry.
  • 502 — Provider is down. The upstream model provider returned an error.
  • 503 — No available provider. OpenRouter couldn't find a provider to serve your request.

Zero-Completion Insurance

OpenRouter does not charge for failed or empty responses. If a model returns an error mid-stream or fails to generate, you aren't billed for that portion. This applies even if partial tokens were transmitted before the failure.

Retry Strategies

  • Exponential backoff — Wait 1s, 2s, 4s, 8s, and so on between retries.
  • Max retries — Set a limit (3-5 attempts) to avoid infinite loops.
  • Jitter — Add random noise to the delay to prevent the thundering herd problem.

Retry-After Header

When you get a 429 response, check the Retry-After header. It tells you exactly how many seconds to wait before your next request. Always prefer this over guessing.

Error Response Format

Errors come back as structured JSON:

{ "error": { "code": 401, "message": "..." } }

Provider Errors

When a provider fails, the error includes metadata about who failed and why. You can find error.metadata.provider_name and error.metadata.raw to debug provider-specific issues.

Lab

Write a robust retry function that handles common errors gracefully:

python3 -c "
import requests, os, time, random

def robust_call(messages, model='openai/gpt-4o-mini:free', max_retries=3):
    url = 'https://openrouter.ai/api/v1/chat/completions'
    headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

    for attempt in range(max_retries):
        resp = requests.post(url, headers=headers, json={'model': model, 'messages': messages})

        if resp.status_code == 200:
            return resp.json()

        if resp.status_code == 429:
            retry_after = int(resp.headers.get('Retry-After', 2 ** attempt))
            print(f'Rate limited. Waiting {retry_after}s...')
            time.sleep(retry_after + random.uniform(0, 1))
            continue

        if resp.status_code in (502, 503):
            wait = (2 ** attempt) + random.uniform(0, 1)
            print(f'Provider error {resp.status_code}. Retrying in {wait:.1f}s...')
            time.sleep(wait)
            continue

        # 401, 402, 403 — don't retry, fix the problem
        error = resp.json().get('error', {})
        raise Exception(f\"{error.get('code', resp.status_code)}: {error.get('message', 'Unknown error')}\")

    raise Exception('Max retries exceeded')

try:
    result = robust_call([{'role': 'user', 'content': 'Hello!'}])
    print(result['choices'][0]['message']['content'])
except Exception as e:
    print(f'Call failed: {e}')
"

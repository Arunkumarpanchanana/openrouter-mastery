Session 05: Understanding API Responses

⏱ 25 min · 🌱 Beginner · API Concepts · 📝 5 Quiz Questions
📖 Learn 🔬 Lab 📝 Quiz

Learn

Response Anatomy

Every chat completion response contains these key fields:

  • id — A unique generation ID. Use this for auditing, debugging, and querying usage stats later.
  • choices — An array of completion objects. Usually contains one element unless you set n > 1.
  • usage — Token counts that determine your bill.
  • model — The model that actually served the request (may differ from what you requested if routing is configured).
  • created — Unix timestamp of when the response was generated.

Choices

Each choice contains:

  • message.content — The actual text generated by the model
  • finish_reason — Why generation stopped. Common values: stop (natural end), length (hit max_tokens), tool_calls (model requested tools), error (something went wrong)
  • native_finish_reason — The provider's original finish reason before normalization

Usage

The usage object tells you:

  • prompt_tokens — Tokens in your input
  • completion_tokens — Tokens in the generated output
  • total_tokens — Sum of both

Cost Calculation

Your actual cost for a request is:

(prompt_tokens × prompt_price) + (completion_tokens × completion_price)

For :free models, this cost is always zero. For paid models, the generation endpoint response may include a cost field in the usage object.

The Generation ID

Save the id field. You can use it later to query generation details via the /api/v1/generation/{id} endpoint, which returns token usage, cost, and other metadata.

Lab

Send a request and parse the full response to understand every field:

python3 -c "
import requests, os

resp = requests.post(
    'https://openrouter.ai/api/v1/chat/completions',
    headers={
        'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}',
        'Content-Type': 'application/json',
    },
    json={
        'model': 'openai/gpt-4o-mini:free',
        'messages': [
            {'role': 'user', 'content': 'Write a haiku about APIs'}
        ],
    },
)

data = resp.json()
choice = data['choices'][0]
usage = data['usage']

print(f'Response ID: {data[\"id\"]}')
print(f'Model used: {data[\"model\"]}')
print(f'Created at: {data[\"created\"]}')
print()
print(f'Content: {choice[\"message\"][\"content\"]}')
print(f'Finish reason: {choice[\"finish_reason\"]}')
print()
print(f'Tokens — Prompt: {usage[\"prompt_tokens\"]}, '\
      f'Completion: {usage[\"completion_tokens\"]}, '\
      f'Total: {usage[\"total_tokens\"]}')
print(f'Cost: \${usage[\"cost\"]:.8f}' if usage.get('cost') else 'Cost: \$0.00 (free model)')
"

Now try a longer generation and observe the length finish reason:

python3 -c "
import requests, os

resp = requests.post(
    'https://openrouter.ai/api/v1/chat/completions',
    headers={
        'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}',
        'Content-Type': 'application/json',
    },
    json={
        'model': 'openai/gpt-4o-mini:free',
        'messages': [
            {'role': 'user', 'content': 'Write a 500-word essay about AI'}
        ],
        'max_tokens': 20,
    },
)

data = resp.json()
choice = data['choices'][0]
usage = data['usage']

print(f'Finish reason: {choice[\"finish_reason\"]} (expected: length)')
print(f'Completion tokens: {usage[\"completion_tokens\"]} (≈ max_tokens=20)')
print(f'Content preview: {choice[\"message\"][\"content\"][:50]}...')
"

Quiz

Q1. What does the id field in the API response represent?

Show answer

b — The id is a unique identifier for that specific generation, useful for auditing and tracing.

Q2. Which finish_reason indicates the model finished generating naturally?

Show answer

b — stop means the model completed its response naturally. length means it hit the max_tokens limit.

Q3. How is the cost of an API call calculated?

Show answer

b — Input and output tokens are priced separately, so you multiply each by its respective rate and sum them.

Q4. What information does the usage object in the response contain?

Show answer

b — The usage object contains the token counts for input, output, and the total.

Q5. What can you do with the generation ID from an API response?

Show answer

b — The generation ID lets you look up detailed metadata about that specific request, including token usage and cost.

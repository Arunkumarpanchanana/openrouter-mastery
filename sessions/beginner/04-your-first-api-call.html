Session 04: Your First API Call

⏱ 35 min · 🌱 Beginner · API Hands-on · 📝 5 Quiz Questions
📖 Learn 🔬 Lab 📝 Quiz

Learn

The Chat Completions Endpoint

The primary endpoint for generating text is POST /api/v1/chat/completions. This is the OpenAI-compatible endpoint that works as a drop-in replacement for the OpenAI API.

Request Structure

A minimal request requires:

  • model — The model ID string (e.g., openai/gpt-4o-mini:free)
  • messages — An array of message objects with role and content
  • Optional parameters — temperature, max_tokens, top_p, etc.

Message Roles

Three roles control the conversation:

  • system — Instructions that set the behavior and persona of the model
  • user — The input or question from the user
  • assistant — Previous responses from the model (used for multi-turn conversations)

Using the OpenAI SDK

Because OpenRouter is OpenAI-compatible, you can use the official OpenAI Python SDK by simply changing the base_url to https://openrouter.ai/api/v1. This means all your existing OpenAI code works with any OpenRouter model.

Authentication

Every request includes your API key in the Authorization: Bearer header. The Content-Type must be application/json.

Response Structure

The response contains:

  • id — Unique generation identifier
  • choices — Array of generated completions
  • usage — Token counts for billing
  • model — Which model actually served the request
  • created — Timestamp

Lab

Make your first API call using Python's requests library:

python3 -c "
import requests
import os

response = requests.post(
    'https://openrouter.ai/api/v1/chat/completions',
    headers={
        'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}',
        'Content-Type': 'application/json',
    },
    json={
        'model': 'openai/gpt-4o-mini:free',
        'messages': [
            {'role': 'user', 'content': 'Say hello in 5 languages'}
        ],
    },
)

print(response.json()['choices'][0]['message']['content'])
"

Now try the same call using the OpenAI SDK (install it first if needed):

pip install openai

python3 -c "
from openai import OpenAI
import os

client = OpenAI(
    base_url='https://openrouter.ai/api/v1',
    api_key=os.environ['OPENROUTER_API_KEY'],
)

response = client.chat.completions.create(
    model='openai/gpt-4o-mini:free',
    messages=[{'role': 'user', 'content': 'Say hello in 5 languages'}],
)
print(response.choices[0].message.content)
"

Try this: Change the model string to anthropic/claude-3-haiku:free or google/gemini-2.0-flash:free and run the same code. Your code works with zero changes — only the model string changes!

Quiz

Q1. What is the base endpoint for chat completions on OpenRouter?

Show answer

Q2. Which message roles are part of a chat completion request?

Show answer

b — Chat completions use system (instructions), user (input), and assistant (previous responses) roles.

Q3. What does setting base_url to https://openrouter.ai/api/v1 do for the OpenAI SDK?

Show answer

b — OpenRouter's OpenAI compatibility means you can use the OpenAI SDK with any model by just changing the base URL.

Q4. What happens when you change the model string in your request?

Show answer

b — Changing the model string is all you need to switch models. The rest of your code stays the same.

Q5. What Content-Type header is required for POST requests to the chat completions endpoint?

Show answer

b — The chat completions endpoint expects JSON, so the Content-Type must be application/json.

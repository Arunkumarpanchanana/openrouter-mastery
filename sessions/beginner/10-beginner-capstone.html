Session 10: Capstone: CLI Chatbot

⏱ 45 min · 🌱 Beginner · 📝 5 Quiz Questions

Learn

Bringing It All Together

This capstone combines everything you've learned: making API calls, tuning parameters, handling errors, streaming responses, and switching models on the fly. You'll build a complete interactive chatbot that runs in your terminal.

Chatbot Architecture

A chatbot loop follows a simple pattern: read user input → call the API → print the response → repeat. The key innovation over a one-shot request is state management — preserving the conversation history across turns so the model remembers context.

State Management

Each turn appends both the user's message and the assistant's response to a messages list. This growing history is sent with every request, giving the model full context. A system message at the start sets the bot's personality and constraints.

Graceful Features

Your chatbot should handle clean exit (a quit command), dynamic model switching without restarting (model:xxx), and runtime parameter changes (temp:0.5). Error recovery ensures one failed request doesn't crash the session.

Lab

Build a complete interactive CLI chatbot with streaming, model switching, and error recovery:

python3 -c "
import requests, os, json, sys

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {
    'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}',
    'Content-Type': 'application/json',
}

def chat_with_model(model='openai/gpt-4o-mini:free', temperature=0.7):
    messages = [{'role': 'system', 'content': 'You are a helpful assistant. Be concise.'}]
    print(f'\n🤖 Chatbot ready! Using model: {model}')
    print(\"Type 'quit' to exit, 'model:xxx' to switch models, 'temp:0.5' to change temperature.\n\")

    while True:
        user_input = input('You: ').strip()

        if user_input.lower() == 'quit':
            print('Goodbye!')
            break
        if user_input.startswith('model:'):
            model = user_input[6:].strip()
            print(f'Switched to: {model}')
            continue
        if user_input.startswith('temp:'):
            temperature = float(user_input[5:].strip())
            print(f'Temperature set to: {temperature}')
            continue

        messages.append({'role': 'user', 'content': user_input})

        try:
            resp = requests.post(url, headers=headers, json={
                'model': model,
                'messages': messages,
                'temperature': temperature,
                'stream': True,
            })

            buffer = ''
            assistant_text = ''
            print('\n🤖 ', end='', flush=True)

            for chunk in resp.iter_content(chunk_size=1024, decode_unicode=True):
                buffer += chunk
                while '\n' in buffer:
                    line, buffer = buffer.split('\n', 1)
                    line = line.strip()
                    if line.startswith('data: '):
                        data = line[6:]
                        if data == '[DONE]':
                            break
                        try:
                            parsed = json.loads(data)
                            content = parsed['choices'][0]['delta'].get('content', '')
                            if content:
                                print(content, end='', flush=True)
                                assistant_text += content
                        except json.JSONDecodeError:
                            pass

            print('\n')
            messages.append({'role': 'assistant', 'content': assistant_text})

        except Exception as e:
            print(f'\nError: {e}')

if __name__ == '__main__':
    chat_with_model()
"

Save this as chatbot.py and run it with python3 chatbot.py for a persistent interactive session. Try switching models mid-conversation with model:google/gemini-2.0-flash-exp:free.

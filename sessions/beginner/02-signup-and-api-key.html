Session 02: Sign Up & API Keys

⏱ 20 min · 🌱 Beginner · Setup Hands-on · 📝 5 Quiz Questions

Learn

Creating an Account

Visit openrouter.ai/keys to get started. You can sign up with your email address or use Google/GitHub OAuth for a faster setup.

API Keys — Your Secret Credential

Your API key is the most sensitive credential in your OpenRouter setup. Treat it like a password — never hardcode it in your source code. The standard practice is to use environment variables.

Security Best Practices

  • Use .env files — Store your key in a .env file in your project root
  • Add .env to .gitignore — Never commit secrets to version control
  • Environment variables — Set OPENROUTER_API_KEY in your shell profile or deployment environment
  • Don't share keys — Each person/application should have its own key
  • Rotate periodically — Generate fresh keys and revoke old ones from the dashboard

Dashboard Overview

The dashboard gives you visibility into:

  • Credits — Your account balance for paid models
  • Usage — Real-time and historical API usage
  • Activity Log — A record of every API call made with your keys

The :free Models

You don't need credits to use :free models. This means you can start learning and building immediately at zero cost. Free models are rate-limited but perfectly adequate for learning and prototyping.

Lab

  1. Go to openrouter.ai/keys and create an account if you haven't already
  2. Generate a new API key and give it a descriptive label (e.g., "learning")
  3. Copy the key and save it to a .env file:
# .env
OPENROUTER_API_KEY=sk-or-v1-your-key-here
  1. Verify the key with curl in your terminal:
export OPENROUTER_API_KEY=sk-or-v1-your-key-here
curl -fsSL https://openrouter.ai/api/v1/auth/key \
  -H "Authorization: Bearer $OPENROUTER_API_KEY"
  1. Verify the key with Python:
python3 -c "
import os
import requests

api_key = os.environ.get('OPENROUTER_API_KEY')
if not api_key:
    raise ValueError('Set OPENROUTER_API_KEY environment variable')

response = requests.get(
    'https://openrouter.ai/api/v1/auth/key',
    headers={'Authorization': f'Bearer {api_key}'}
)
print(response.json())
"

If everything is set up correctly, you'll see your key information in JSON format. If authentication fails, double-check that the key is correctly copied and the environment variable is set.

Session 01: What is OpenRouter?

⏱ 25 min · 🌱 Beginner · Concepts Overview · 📝 5 Quiz Questions
📖 Learn 🔬 Lab 📝 Quiz

Learn

What is OpenRouter?

OpenRouter is a unified API gateway that provides access to 400+ models from dozens of providers — OpenAI, Anthropic, Google, Meta, Mistral, DeepSeek, and many more. Instead of signing up for 10 different APIs with 10 different billing systems and 10 different SDKs, you get one account, one API key, one billing system.

Why It Exists

The AI model landscape is exploding. New models launch weekly. OpenRouter solves four key problems:

  • Fragmentation — N different APIs to learn and integrate
  • Vendor lock-in — Get stuck with a single provider's ecosystem
  • Cost optimization — Hard to compare prices across providers
  • Availability — Individual providers go down; OpenRouter routes around failures

Model Flexibility as a Cost Lever

You can swap models by changing one string in your code. When a cheaper or better model launches tomorrow, you can try it instantly without rewriting anything.

How Routing Works

Your request goes to OpenRouter → the best available provider is selected → the response comes back. You don't care which GPU or data center ran it. The abstraction is complete.

The API Key

Your account identifier. All requests are authenticated via a Bearer token in the Authorization header. Every API key is associated with your account and can be managed from the dashboard.

Free Tier

OpenRouter offers :free model variants — zero-cost inference on select community models. This is perfect for learning, prototyping, and experimentation without spending a cent.

Lab

  1. Visit openrouter.ai
  2. Create an account (or sign in if you already have one)
  3. Explore the dashboard to get familiar with the layout
  4. Find the API Keys page in your account settings
  5. Browse the Models page to see the ecosystem of available models
  6. Verify your API key works with curl:
curl -fsSL https://openrouter.ai/api/v1/auth/key \
  -H "Authorization: Bearer $OPENROUTER_API_KEY"

If your key is set correctly, you'll receive a JSON response with your key details. If not, you'll see an authentication error.

Quiz

Q1. What does OpenRouter do?

Show answer

b — OpenRouter aggregates 400+ models behind a single API, so you use one key and one billing system.

Q2. Which of the following is NOT a problem OpenRouter solves?

Show answer

c — OpenRouter doesn't handle model training. It provides access to already-trained models from various providers.

Q3. How do you authenticate requests to OpenRouter?

Show answer

b — OpenRouter uses Bearer token authentication via the Authorization header with your API key.

Q4. How does model routing work in OpenRouter?

Show answer

b — OpenRouter handles provider selection and routing behind the scenes. You just specify the model name.

Q5. What is the :free model variant in OpenRouter?

Show answer

b — The :free suffix indicates community-hosted models available for inference at zero cost.

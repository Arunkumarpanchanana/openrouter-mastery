Session 06: Parameters Deep Dive

⏱ 30 min · 🌱 Beginner · 📝 5 Quiz Questions

Learn

temperature

Range: 0.0 – 2.0 | Default: 1.0

Controls randomness in the output. Low values (near 0) make the model deterministic and focused — same input always produces the same output. High values (near 2.0) make it creative and varied. Use low temperature for factual tasks like classification or data extraction. Use high temperature for creative writing, brainstorming, or generating multiple ideas.

At temperature 0, the model becomes effectively deterministic: the same prompt with the same parameters will return the same result every time.

max_tokens

A hard limit on the length of the response. The model will stop generating once this limit is reached, and the finish_reason will be length. If the model finishes naturally before the limit, generation stops early. You are only charged for the tokens actually generated, not the max_tokens value.

top_p

Range: 0.0 – 1.0 | Default: 1.0

Also known as nucleus sampling. Instead of considering all possible next tokens, the model considers only the smallest set of tokens whose cumulative probability reaches top_p. A value of 0.1 means only the top 10% most likely tokens are considered. This is an alternative to temperature — typically you adjust one or the other, not both.

seed

Type: Integer

When set, the model will make a best-effort attempt to return deterministic results. The same seed combined with the same parameters should produce the same output. This is a best-effort guarantee — not all models support it fully, and provider-side changes can affect reproducibility.

stop

Type: String or Array of strings

Stop sequences tell the model to halt generation when it encounters specific text. For example, stop: ["###"] will stop generation as soon as the model outputs "###". Useful for structured outputs where you want generation to end at a delimiter.

frequency_penalty

Range: -2.0 – 2.0

Penalizes tokens based on how frequently they've appeared in the text so far. Higher values reduce repetition by making the model less likely to reuse the same words and phrases.

presence_penalty

Range: -2.0 – 2.0

Penalizes tokens that have appeared at all in the text, regardless of frequency. Higher values encourage the model to talk about new topics and introduce more diversity.

Lab

Experiment with parameters to see how they change the model's output:

python3 -c "
import requests, os

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

prompt = 'Give me one creative startup idea in one sentence.'

# Temperature comparison
for temp in [0.0, 0.7, 1.5]:
    resp = requests.post(url, headers=headers, json={
        'model': 'openai/gpt-4o-mini:free',
        'messages': [{'role': 'user', 'content': prompt}],
        'temperature': temp,
        'max_tokens': 100,
    })
    print(f'\n--- Temperature {temp} ---')
    print(resp.json()['choices'][0]['message']['content'])

# Seed determinism
print('\n\n--- Seed 42 (run twice) ---')
for i in range(2):
    resp = requests.post(url, headers=headers, json={
        'model': 'openai/gpt-4o-mini:free',
        'messages': [{'role': 'user', 'content': 'Pick a random number between 1 and 100 and explain why.'}],
        'seed': 42,
        'temperature': 0.7,
        'max_tokens': 100,
    })
    print(f'Run {i+1}: {resp.json()[\"choices\"][0][\"message\"][\"content\"][:80]}...')
"

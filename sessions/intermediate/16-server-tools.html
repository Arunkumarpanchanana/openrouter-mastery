Session 16: Server Tools

⏱ 35 min · 💻 Intermediate · 📝 5 Quiz Questions

Learn

What Are Server Tools?

Server tools are built-in capabilities that run on OpenRouter's side — no code, no agent loop, no infrastructure needed. You just add a plugins array to your request and the model can use them automatically.

Available Server Tools

  • web — The model can search the internet and return results with citations. Add plugins: [{"id": "web"}] to your request body.
  • web_search — Legacy web search access (prefer web plugin for new implementations).
  • fetch — The model can fetch content from a specific URL and use it as context for its response.
  • datetime — The model knows the current date and time, useful for time-sensitive queries like "what events happened today?"

Why Server Tools Matter

Server tools eliminate the need to write an agent loop. You don't need to define tool schemas, handle tool_calls, execute functions, or manage state. The model gets the capability built-in and uses it transparently. This works with any compatible model and requires zero infrastructure on your side.

Plugin System

The plugins array in your request body enables these capabilities. Simply set plugins: [{"id": "web"}, {"id": "datetime"}] to give the model access to both tools in a single request.

Lab

Use the web search plugin to get real-time information:

python3 -c "
import requests, os

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

# Web search with plugin
resp = requests.post(url, headers=headers, json={
    'model': 'openai/gpt-4o-mini:free',
    'messages': [{'role': 'user', 'content': 'What are the latest developments in AI this month? Search the web and cite sources.'}],
    'plugins': [{'id': 'web'}],
})

data = resp.json()
print(data['choices'][0]['message']['content'])
"

Now test datetime awareness:

python3 -c "
import requests, os

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

resp = requests.post(url, headers=headers, json={
    'model': 'openai/gpt-4o-mini:free',
    'messages': [{'role': 'user', 'content': \"What's today's date and what day of the week is it?\"}],
    'plugins': [{'id': 'datetime'}],
})

print(resp.json()['choices'][0]['message']['content'])
"

Try this: Combine both plugins in a single request — ask for "news from today" and let the model use both web search and datetime together.

Session 21: Capstone: Multi-Tool Research Agent

⏱ 50 min · 💻 Intermediate · 📝 5 Quiz Questions

Learn

Bringing It All Together

This capstone combines everything from the intermediate track: streaming, tools, structured outputs, message history management, and error handling — all in a single multi-tool agent.

Multi-Tool Agents

Instead of defining one tool, give the model multiple tools and let it choose which to use based on the task. The model reasons about what information it needs and selects the appropriate tool, just like a human would.

The Research Agent Pattern

The classic research agent flow is: user asks a question → the model searches sources (Wikipedia, web) → extracts structured data (calculations, lookups) → synthesizes everything into a coherent answer.

Design Considerations

  • Tool descriptions must be clear — The model chooses tools based on their descriptions. Vague descriptions lead to wrong tool selection.
  • Message history must be preserved — Every tool call, result, and model turn must be appended to the messages array.
  • Error isolation — A failure in one tool shouldn't crash the entire loop. Wrap tool execution in try/except and return meaningful error messages to the model so it can recover.
  • Tool routing — Use a dispatch map to route function.name strings to Python functions cleanly.

Lab

Build a research agent with multiple tools — Wikipedia search, math calculation, and time lookup:

python3 << 'EOF'
import requests, json, os, time

url = "https://openrouter.ai/api/v1/chat/completions"
headers = {"Authorization": f"Bearer {os.environ['OPENROUTER_API_KEY']}"}

def search_wikipedia(query):
    import urllib.parse
    r = requests.get(f"https://en.wikipedia.org/api/rest_v1/page/summary/{urllib.parse.quote(query)}")
    if r.status_code == 200:
        d = r.json()
        return {"title": d["title"], "extract": d["extract"][:800]}
    return {"error": "Not found"}

def calculate(expression):
    try:
        return {"result": eval(expression)}
    except:
        return {"error": "Invalid expression"}

def get_time():
    return {"current_time": time.strftime("%Y-%m-%d %H:%M:%S UTC", time.gmtime())}

tools = [
    {"type": "function", "function": {
        "name": "search_wikipedia", "description": "Search Wikipedia for a topic",
        "parameters": {"type": "object", "properties": {"query": {"type": "string"}}, "required": ["query"]
    }}},
    {"type": "function", "function": {
        "name": "calculate", "description": "Evaluate a mathematical expression",
        "parameters": {"type": "object", "properties": {"expression": {"type": "string"}}, "required": ["expression"]
    }}},
    {"type": "function", "function": {
        "name": "get_time", "description": "Get the current date and time",
        "parameters": {"type": "object", "properties": {}}
    }},
]

TOOL_MAP = {"search_wikipedia": search_wikipedia, "calculate": calculate, "get_time": get_time}

messages = [{"role": "user", "content": "When was Albert Einstein born? Also calculate how many years ago that was from today."}]
MAX_ITER = 5

for i in range(MAX_ITER):
    resp = requests.post(url, headers=headers, json={
        "model": "openai/gpt-4o-mini:free", "messages": messages, "tools": tools
    })
    msg = resp.json()["choices"][0]["message"]
    messages.append(msg)

    if msg.get("tool_calls"):
        for tc in msg["tool_calls"]:
            fn_name = tc["function"]["name"]
            fn_args = json.loads(tc["function"]["arguments"])
            print(f"  Calling {fn_name}({fn_args})")
            result = TOOL_MAP[fn_name](**fn_args)
            messages.append({"role": "tool", "tool_call_id": tc["id"], "content": json.dumps(result)})
    else:
        print(f"\nFinal answer:\n{msg['content']}")
        break
EOF

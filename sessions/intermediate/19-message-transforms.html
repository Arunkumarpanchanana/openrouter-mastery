Session 19: Message Transforms

⏱ 30 min · 💻 Intermediate · 📝 5 Quiz Questions
📖 Learn 🔬 Lab 📝 Quiz

Learn

The Problem

Different models have different context windows. Some models can't handle huge conversation histories. Long-running agent sessions accumulate messages, and eventually you exceed the context limit or pay high costs for processing all those tokens.

What Are Message Transforms?

Message transforms are OpenRouter's built-in mechanism to compress or optimize messages before sending them to the provider. The model provider never sees the full raw history — they receive a transformed version that fits within their context window.

Middle-Out Compression

The context-compression plugin keeps the system prompt and most recent messages intact while compressing the middle of the conversation. This preserves critical context (instructions and recent exchanges) while reducing token count.

Key Options

  • min_chars — Minimum characters to keep from compressed sections
  • keep_system — Preserve the system prompt (default: true)
  • max_tokens — Target output size after compression

When to Use

Message transforms are ideal for long-running agents with accumulating history, models with smaller context windows, and cost-sensitive applications where you want to minimize token usage.

How to Enable

Add plugins: [{"id": "context-compression"}] to your request body. The compression happens automatically on OpenRouter's side before forwarding to the provider.

Trade-Offs

Compression loses some detail from the middle of conversations. The compressed summary may not capture every nuance, but it prevents truncation errors (where the model loses the end of the conversation entirely).

Lab

Enable message transforms and compare behavior with and without compression:

python3 << 'EOF'
import requests, os

url = "https://openrouter.ai/api/v1/chat/completions"
headers = {"Authorization": f"Bearer {os.environ['OPENROUTER_API_KEY']}"}

# Build a long conversation
messages = [{"role": "system", "content": "You are a helpful assistant."}]
for i in range(10):
    messages.append({"role": "user", "content": f"This is message number {i}. Please acknowledge each one."})
    messages.append({"role": "assistant", "content": f"Acknowledged message {i}."})

# Without transforms
resp1 = requests.post(url, headers=headers, json={
    "model": "openai/gpt-4o-mini:free",
    "messages": messages + [{"role": "user", "content": "How many messages did I send?"}],
})
print("Without transforms:", resp1.json()["choices"][0]["message"]["content"][:200])

# With context compression plugin
resp2 = requests.post(url, headers=headers, json={
    "model": "openai/gpt-4o-mini:free",
    "messages": messages + [{"role": "user", "content": "How many messages did I send?"}],
    "plugins": [{"id": "context-compression"}],
})
print("\nWith transforms:", resp2.json()["choices"][0]["message"]["content"][:200])
EOF

Session 18: Building Agent Patterns

⏱ 45 min · 💻 Intermediate · 📝 5 Quiz Questions

Learn

The Agent Loop

The core agent pattern is a loop: send messages to the model → the model requests a tool → execute the tool → send the result back → the model responds — repeat if the model requests more tools, or break when it provides a final answer.

Why Agent Patterns?

Agents enable autonomous task completion, multi-step reasoning, and external data access. Instead of a single Q&A exchange, the model can search, calculate, fetch, and synthesize across multiple turns.

State Management

Maintaining full message history across turns is critical. Every tool call, tool result, and model response must be appended to messages. Without complete history, the model loses context and produces worse results on subsequent turns.

Max Iterations Guard

Always set max_iterations (typically 10-20) to prevent infinite loops. Bugs in tool definitions or unexpected model behavior can cause the model to call tools forever without producing a final answer.

Agent Patterns

  • Simple tool call — One-and-done: model calls a tool once, gets result, produces answer
  • Iterative refinement — Model calls the same tool multiple times with different parameters to build up context
  • Parallel tool execution — Model requests multiple tools in a single response (batch tool_calls)
  • Conditional branching — Model decides which tool to call based on previous results

The Full Pattern

System prompt + tool definitions + complete message history = powerful autonomous behavior. The three components work together: the system prompt sets the agent's persona and instructions, tools define what it can do, and message history provides context across turns.

Lab

Build a complete agent loop that handles multiple tool calls with Wikipedia search:

python3 << 'EOF'
import requests, json, os

url = "https://openrouter.ai/api/v1/chat/completions"
headers = {"Authorization": f"Bearer {os.environ['OPENROUTER_API_KEY']}"}

def search_wikipedia(query):
    import urllib.parse
    q = urllib.parse.quote(query)
    r = requests.get(f"https://en.wikipedia.org/api/rest_v1/page/summary/{q}")
    if r.status_code == 200:
        data = r.json()
        return {"title": data["title"], "extract": data["extract"][:500]}
    return {"error": "Not found"}

tools = [{
    "type": "function",
    "function": {
        "name": "search_wikipedia",
        "description": "Search Wikipedia for a topic and get summary",
        "parameters": {
            "type": "object",
            "properties": {"query": {"type": "string", "description": "Topic to search"}},
            "required": ["query"]
        }
    }
}]

messages = [{"role": "user", "content": "Research the history of artificial intelligence and summarize key milestones."}]
MAX_ITER = 5

for i in range(MAX_ITER):
    resp = requests.post(url, headers=headers, json={
        "model": "openai/gpt-4o-mini:free",
        "messages": messages, "tools": tools
    })
    msg = resp.json()["choices"][0]["message"]
    messages.append(msg)

    if msg.get("tool_calls"):
        for tc in msg["tool_calls"]:
            args = json.loads(tc["function"]["arguments"])
            result = search_wikipedia(args["query"])
            messages.append({"role": "tool", "tool_call_id": tc["id"], "content": json.dumps(result)})
    else:
        print(f"Final answer (after {i+1} rounds):\n{msg['content']}")
        break
else:
    print("Max iterations reached")
EOF

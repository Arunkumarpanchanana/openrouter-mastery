Session 11: Structured Outputs & JSON Schema

⏱ 35 min · 💻 Intermediate · 📝 5 Quiz Questions

Learn

The Problem with Unstructured Text

LLMs return unstructured text by default. Parsing it programmatically is fragile — one typo, slightly different phrasing, or an unexpected format can break your entire pipeline. JSON parsing errors, regex failures, and silent data corruption are common when you try to extract structured data from natural language output.

The response_format Parameter

OpenRouter's response_format parameter forces the model to return valid JSON. It has two modes:

  • json_object — The model returns any valid JSON. You define no schema; the model decides the structure based on the prompt.
  • json_schema — You provide an exact JSON Schema definition and strict: true guarantees the output matches it perfectly.

JSON Schema Definition

A JSON Schema defines the shape of your data. Key fields include:

  • type — Always "object" at the top level
  • properties — Each field with its own type (string, number, array, etc.) and an optional description
  • required — Array of field names that must be present
  • additionalProperties — Set to false to prevent extra fields
  • strict: true — Guarantees the output matches the schema exactly

When to Use Structured Outputs

Use this feature for: extraction pipelines (pulling data from documents), data processing (normalizing messy input), API integrations (generating structured payloads), form filling, database inserts, and any workflow where downstream code depends on consistent field names and types.

Model Support

Check supported_parameters for the model to see if it supports "structured_outputs". Use require_parameters: true in your provider config to ensure only compatible providers handle your request. Not all models support structured outputs.

Lab

Extract structured financial data from unstructured text using JSON schema:

python3 -c "
import requests, os, json

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

text = 'Apple Inc. reported Q3 revenue of \$85.8 billion on August 1st, 2024. Tim Cook announced the results.'

response = requests.post(url, headers=headers, json={
    'model': 'openai/gpt-4o-mini:free',
    'messages': [{'role': 'user', 'content': f'Extract financial data from: {text}'}],
    'response_format': {
        'type': 'json_schema',
        'json_schema': {
            'name': 'financial_data',
            'strict': True,
            'schema': {
                'type': 'object',
                'properties': {
                    'company': {'type': 'string', 'description': 'Company name'},
                    'revenue_billions': {'type': 'number', 'description': 'Revenue in billions'},
                    'quarter': {'type': 'string', 'description': 'Fiscal quarter'},
                    'date': {'type': 'string', 'description': 'Announcement date'},
                    'ceo': {'type': 'string', 'description': 'CEO name'}
                },
                'required': ['company', 'revenue_billions', 'quarter', 'date'],
                'additionalProperties': False
            }
        }
    }
})

data = response.json()
result = json.loads(data['choices'][0]['message']['content'])
print(json.dumps(result, indent=2))
print(f\"\\nCompany: {result['company']}, Revenue: \${result['revenue_billions']}B\")
"

Try this: Modify the schema to extract additional fields like stock ticker or net income. Change the input text to a different company's earnings report and see how the extraction handles it.

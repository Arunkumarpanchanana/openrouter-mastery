Session 32: Frameworks & Ecosystem

⏱ 35 min · 🚀 Expert · 📝 5 Quiz Questions

Learn

OpenAI Compatibility

OpenRouter is OpenAI-compatible — ANY framework that works with OpenAI works with OpenRouter. The only changes needed are swapping the base_url and api_key. This means thousands of existing tools, libraries, and frameworks work with OpenRouter out of the box.

LangChain

Use OpenRouter with LangChain by setting openai_api_base="https://openrouter.ai/api/v1" on the ChatOpenAI class. All LangChain integrations — chains, agents, tools, retrievers — work without modification. This is the quickest way to add multi-model routing to existing LangChain applications.

Vercel AI SDK

Configure the provider as a custom OpenAI-compatible provider with OpenRouter's base URL. The Vercel AI SDK's streaming, tool calling, and generation features all work seamlessly. This is the standard approach for Next.js applications.

PydanticAI & Anthropic Agent SDK

PydanticAI provides a Python framework for AI agents with structured outputs. Set base_url to OpenRouter for multi-model access. The Anthropic Agent SDK also works with OpenRouter for building agents in both Python and TypeScript — giving you access to OpenRouter's model ecosystem through your preferred agent framework.

MCP Server Integration

Connect MCP (Model Context Protocol) tools through OpenRouter for model access. This lets you use OpenRouter's routing and model selection with MCP-compatible tool ecosystems, combining structured tool access with flexible model selection.

Community Integrations

OpenRouter integrates with Zapier (8000+ app automations), Replit (bring your own key), LiveKit (voice AI), and the Effect AI SDK. The community-maintained "Awesome OpenRouter" list catalogs hundreds of projects and apps built on OpenRouter.

Lab

View the configuration patterns for popular frameworks:

python3 -c "
import os

# This shows the configuration pattern (requires langchain-openai package)
config_code = '''
from langchain_openai import ChatOpenAI

# Same OpenAI interface, different base_url
llm = ChatOpenAI(
    model=\"openai/gpt-4o-mini:free\",
    openai_api_key=os.environ[\"OPENROUTER_API_KEY\"],
    openai_api_base=\"https://openrouter.ai/api/v1\",
    default_headers={
        \"HTTP-Referer\": \"https://openroutermastery.dev\",
        \"X-OpenRouter-Title\": \"OpenRouter Mastery\",
    }
)

response = llm.invoke(\"What is LangChain?\")
print(response.content)
'''
print('LangChain integration pattern:\n')
print(config_code)

# Show Vercel AI SDK pattern
vercel_code = '''
// Vercel AI SDK with OpenRouter
import { createOpenAI } from '@ai-sdk/openai';

const openrouter = createOpenAI({
  baseURL: \"https://openrouter.ai/api/v1\",
  apiKey: process.env.OPENROUTER_API_KEY,
});

const model = openrouter(\"openai/gpt-4o-mini:free\");
const response = await generateText({ model, prompt: \"Hello!\" });
'''
print('\nVercel AI SDK pattern:\n')
print(vercel_code)
"

Try this: If you have LangChain installed, run the actual code to verify that OpenRouter responses flow through LangChain's interface identically to OpenAI. The key insight is that no framework modifications are needed — just the base URL change.

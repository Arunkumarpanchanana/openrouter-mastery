Session 30: Advanced Server Tools

⏱ 40 min · 🚀 Expert · 📝 5 Quiz Questions
📖 Learn 🔬 Lab 📝 Quiz

Learn

Advisor Tool

Let a fast/cheap model consult a smarter model when uncertain. This implements the FrugalGPT/RouteLLM pattern — a cheap model handles most requests, and only escalates to an expensive model when confidence is low. Zero orchestration code needed on your end; OpenRouter manages the conversation between models.

Subagent Tool

A large model breaks work into subtasks and delegates to smaller, cheaper worker models. Built-in task decomposition without you writing any orchestration logic. The parent model plans the work, the subagents execute, and results are assembled automatically.

Fusion Tool

A panel of models plus a judge. Multiple models each respond to the same prompt, then a judge model synthesizes the best answer from all responses. This delivers higher quality through deliberation — different models catch different errors and the judge selects the strongest output.

Apply Patch Tool

A model proposes V4A diff patches for file edits. OpenRouter validates them before applying. This enables agentic code editing workflows where models propose changes and the server validates correctness before execution.

Opt-In Server Tools

All server tools are opt-in via the tools array or plugins array in your request. You choose which capabilities to enable for each request, keeping your integration clean and intentional.

Server vs. Client Tools

Server tools run on OpenRouter's infrastructure — no server management needed on your end. This differs from client tools which run in your application. Server tools are zero-infrastructure: you just configure them in the API request and OpenRouter handles execution.

Lab

Use the Fusion pattern via OpenRouter's server tools to get a deliberated answer from multiple models:

python3 -c "
import requests, os

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

# Fusion router — multiple models deliberate
resp = requests.post(url, headers=headers, json={
    'model': 'openrouter/fusion',
    'messages': [{'role': 'user', 'content': 'What are the pros and cons of renewable energy? Give a balanced analysis.'}],
    'max_tokens': 500,
})
data = resp.json()
print('Fusion response:')
print(data['choices'][0]['message']['content'])
print(f'\nModel used: {data.get(\"model\", \"N/A\")}')
"

Try this: Compare a Fusion response to a single-model response for the same prompt. Notice how the Fusion output incorporates perspectives from multiple models, often producing more balanced and nuanced answers.

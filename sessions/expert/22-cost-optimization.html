Session 22: Cost Optimization Architecture

⏱ 40 min · 🚀 Expert · 📝 5 Quiz Questions

Learn

The Core Insight

Not every step in your pipeline needs a frontier model. The key to cost optimization is breaking your workflow into discrete steps and assigning the right model to each. A classification step that maps input to one of five categories doesn't need Claude Sonnet — a $0.15/M token model works just as well.

Workflow Decomposition

Identify which steps need reasoning (use expensive models) and which need pattern matching or fast processing (use cheap models). For example:

  • Intent classification → free/mini model
  • Entity extraction → cheap model with structured outputs
  • Complex reasoning → frontier model (Claude, GPT-4o)
  • Response formatting → cheap model or template

Total Cost Calculation

The formula is simple: (prompt_tokens × prompt_price) + (completion_tokens × completion_price). Both sides matter — a model with cheap prompt pricing but expensive completion pricing may cost more than expected if your use case generates long responses. Always calculate total cost, not just per-token rates.

Model Tier Strategy

Classify tasks into tiers and assign models accordingly:

  • Simple: Free models or $0.15/M tokens — classification, extraction, formatting
  • Medium: GPT-4o-mini class — summarization, structured generation
  • Complex: Claude Sonnet / GPT-4o class — reasoning, analysis, code generation

Provider Sorting & Limits

Use provider.sort: "price" with max_price limits to cap costs automatically. OpenRouter will try providers in ascending price order. The :free and :floor variants route to the cheapest available endpoints.

The 80/20 Rule

80% of your costs likely come from 20% of your requests. Profile your usage and optimize those expensive paths first. A single unoptimized reasoning chain can cost more than hundreds of cheap classification calls.

Batch APIs

For non-latency-sensitive work, use batch APIs to get a flat discount. Send multiple requests at once and get results asynchronously — ideal for data processing pipelines, content generation at scale, and backfills.

Lab

Profile a multi-step pipeline and analyze its costs:

python3 -c "
import requests, os, json

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

# Get pricing for different models
resp = requests.get('https://openrouter.ai/api/v1/models')
models_data = {m['id']: m for m in resp.json()['data']}

models_to_check = [
    'openai/gpt-4o-mini:free',
    'openai/gpt-4o',
    'anthropic/claude-sonnet-4',
]

for model_id in models_to_check:
    m = models_data.get(model_id)
    if m:
        prompt_price = float(m['pricing']['prompt']) * 1_000_000
        completion_price = float(m['pricing']['completion']) * 1_000_000
        print(f'{model_id}: \${prompt_price:.4f}/M input, \${completion_price:.4f}/M output, {m.get(\"context_length\", \"?\")} context')

# Simulate cost for a pipeline
pipeline_steps = [
    {'name': 'Classify intent', 'model': 'openai/gpt-4o-mini:free', 'prompt_tokens': 500, 'completion_tokens': 50},
    {'name': 'Extract entities', 'model': 'openai/gpt-4o-mini:free', 'prompt_tokens': 800, 'completion_tokens': 100},
    {'name': 'Generate response', 'model': 'openai/gpt-4o', 'prompt_tokens': 2000, 'completion_tokens': 500},
]

total_cost = 0
for step in pipeline_steps:
    m = models_data.get(step['model'])
    if m:
        cost = (step['prompt_tokens'] * float(m['pricing']['prompt']) +
                step['completion_tokens'] * float(m['pricing']['completion']))
        total_cost += cost
        print(f'  {step[\"name\"]}: \${cost:.6f}')

print(f'\nTotal pipeline cost: \${total_cost:.6f}')
"

Try this: Add a fourth step that uses a cheap model for response formatting. Experiment with different model assignments and see how much you can save by moving simple steps to cheaper models.

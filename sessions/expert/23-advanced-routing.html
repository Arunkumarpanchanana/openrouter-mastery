Session 23: Advanced Routing

⏱ 40 min · 🚀 Expert · 📝 5 Quiz Questions

Learn

Auto Router (NotDiamond)

The Auto Router uses NotDiamond's routing engine to automatically select the best model for each prompt. You don't pick a model — you set "openrouter/auto" as the model and the router analyzes your prompt to choose the optimal model based on capability and cost. This is ideal for production systems where prompt complexity varies widely.

Fusion Router

Fusion is a multi-model deliberation approach. A panel of models independently responds to your prompt, then a judge model synthesizes the best answer from all responses. All of this happens in a single API call. Use model slug "openrouter/fusion". This is powerful for tasks where accuracy matters more than latency — the panel catches errors that any single model might miss.

Pareto Router

The Pareto Router routes coding requests based on a minimum coding score. It filters models by a benchmark threshold, ensuring only models with proven coding ability handle your code-related prompts. Use this when code quality is non-negotiable but you still want cost efficiency.

Free Models Router

The Free Models Router automatically routes to available free models. Use slug "openrouter/free". It checks which free models are currently available and routes to them, falling back if needed. Great for development, testing, and low-priority workloads.

Body Builder

Body Builder lets you use natural language to generate multiple parallel API requests. Describe what you want in plain English and OpenRouter creates the structured requests. This is useful for A/B testing, multi-model comparison, and parallel processing workflows.

Latest Model Resolution

Use ~author/family-latest slugs to auto-route to the newest model in a family. For example, ~openai/gpt-4o-latest always points to the newest GPT-4o variant. This removes the need to update model IDs every time a provider releases a new version.

Partition Sorting

When using model fallbacks, partition: "none" sorts across ALL models and endpoints globally rather than within each partition. This gives you the best overall price or performance across all available options instead of the best within a single provider group.

Lab

Compare different routing strategies side by side:

python3 -c "
import requests, os

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

routers = [
    {'name': 'Free Router', 'model': 'openrouter/free'},
    {'name': 'Auto Router', 'model': 'openrouter/auto'},
    {'name': 'Manual: cheapest', 'model': 'openai/gpt-4o-mini:free'},
]

for router in routers:
    resp = requests.post(url, headers=headers, json={
        'model': router['model'],
        'messages': [{'role': 'user', 'content': 'Explain quantum computing in 2 sentences.'}],
        'max_tokens': 100,
    })
    data = resp.json()
    print(f\"\n{'='*40}\")
    print(f\"Router: {router['name']}\")
    print(f\"Selected model: {data.get('model', 'N/A')}\")
    print(f\"Response: {data['choices'][0]['message']['content'][:100]}...\")
"

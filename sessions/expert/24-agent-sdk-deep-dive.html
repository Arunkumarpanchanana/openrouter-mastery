Learn

OpenRouter Agent SDK

The @openrouter/agent SDK is a TypeScript library for building production-grade agents. It provides core abstractions for model interaction, tool execution, streaming, and loop control — all with first-class TypeScript support.

callModel

callModel is the core function that handles text generation, streaming, and tool calling with automatic execution. You provide messages, tools, and stop conditions — the SDK manages the loop, streams responses, executes tool calls, and feeds results back to the model automatically.

Items-Based Streaming

The items-based streaming paradigm is unique to the Agent SDK. Instead of accumulating chunks of text, you replace items by ID. Each item (a message, a tool result, a status update) has a unique ID. When new data arrives for that ID, the old content is replaced. This simplifies React state management enormously — no more merging partial chunks or tracking cursor positions.

Stop Conditions

The stopWhen helper lets you define conditions to halt agent execution: stop by step count, number of tool calls, accumulated cost, or token budget. No more infinite loops or runaway costs. You can combine conditions — stop after 10 steps OR after \$0.05 in costs, whichever comes first.

nextTurnParams

nextTurnParams enables building encapsulated, context-aware tools. Each turn can pass parameters to the next, creating a chain of context. This is the foundation for plugin systems, adaptive agents that change behavior based on previous turns, and multi-step workflows that maintain state across interactions.

Tool Types

The SDK supports multiple tool types:

  • Regular tools — Define with Zod schema for type-safe arguments
  • Generator tools — Async generators that yield progress updates
  • Manual tools — Human-in-the-loop (HITL) tools that pause and wait for approval
  • Multi-turn tools — Tools that span multiple agent turns, collecting information progressively

Tool Approval

Human-in-the-loop gates for sensitive operations. Define which tools require manual approval before execution. When the model calls an approval-gated tool, execution pauses and waits for a human to confirm. This is critical for production systems that need guardrails around dangerous or expensive operations.

Lab

Explore the Agent SDK's items-based streaming pattern — how it differs from traditional streaming:

The key difference is in how state is managed:

// Traditional streaming — accumulate chunks
let fullContent = '';
stream.onChunk((chunk) => {
  fullContent += chunk;
  setState(prev => prev + chunk);
});

// Items-based streaming — replace by ID
const items = {};

stream.onItem((item) => {
  items[item.id] = item.content;  // Replace, don't append
  setState({ ...items });         // React re-renders cleanly
});

A callModel example showing the core concept:

// Conceptual TypeScript example
import { callModel, stopWhen } from '@openrouter/agent';

const result = await callModel({
  model: 'openai/gpt-4o',
  messages: [{ role: 'user', content: 'Research and summarize the latest AI news.' }],
  tools: [searchWeb, extractArticle],
  stopConditions: [
    stopWhen.maxSteps(5),
    stopWhen.maxCost(0.02),
  ],
  onItem: (item) => {
    // Each item has an id — React state updates are trivial
    updateUI(item.id, item.content);
  },
});

Key insight: Items-based streaming means your UI never needs to merge partial updates. Each item ID maps to a UI element. When the stream sends a new version, React just replaces that element. This eliminates an entire class of rendering bugs.

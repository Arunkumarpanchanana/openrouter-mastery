Session 27: Broadcast & Observability

⏱ 35 min · 🚀 Expert · 📝 5 Quiz Questions

Learn

What is Broadcast?

Broadcast is OpenRouter's observability pipeline. It sends traces of every API call to your chosen observability platform with zero code changes. Flip a switch in the dashboard and start receiving latency, token usage, cost, model selection, provider info, and error data — all without modifying a single line of your application code.

Supported Platforms

OpenRouter Broadcast integrates with a wide range of platforms: Langfuse, Datadog, Braintrust, ClickHouse, Comet Opik, Grafana Cloud, LangSmith, New Relic, PostHog, Sentry, S3 (for long-term archival), Snowflake, Webhook (for custom pipelines), and more. This makes it easy to fit into your existing observability stack regardless of your platform preference.

What You Get

Every trace includes: request and response timestamps (latency), token counts (prompt and completion), cost breakdown, model selected, provider that handled the request, any errors or retries, and guardrail events. This data is essential for debugging, cost analysis, and performance monitoring.

Debug Mode

Enable debug: { echo_upstream_body: true } to see exactly what request body OpenRouter sent to the upstream provider. This is invaluable for troubleshooting provider-specific issues, understanding how OpenRouter transforms your request, and verifying that your parameters are being passed correctly.

Router Metadata

Opt in to receive metadata about the routing decision: which provider was selected, what fallbacks were attempted, and which pipeline stages were executed. This gives you full visibility into the routing layer, helping you understand why a particular provider was chosen and whether fallbacks were triggered.

Lab

Enable debug mode to inspect the transformed request sent to the provider:

python3 -c "
import requests, os, json

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

resp = requests.post(url, headers=headers, json={
    'model': 'openai/gpt-4o-mini:free',
    'messages': [{'role': 'user', 'content': 'Hello!'}],
    'stream': True,
    'debug': {'echo_upstream_body': True},
})

buffer = ''
for chunk in resp.iter_content(chunk_size=1024, decode_unicode=True):
    buffer += chunk
    while '\n' in buffer:
        line, buffer = buffer.split('\n', 1)
        line = line.strip()
        if line.startswith('data: '):
            data = line[6:]
            if data == '[DONE]':
                break
            try:
                parsed = json.loads(data)
                if parsed.get('debug', {}).get('echo_upstream_body'):
                    print('Transformed request body sent to provider:')
                    print(json.dumps(parsed['debug']['echo_upstream_body'], indent=2))
                content = parsed.get('choices', [{}])[0].get('delta', {}).get('content', '')
                if content:
                    print(content, end='', flush=True)
            except:
                pass
"

Try this: Add different model parameters (temperature, max_tokens, response_format) and observe how the transformed request body changes. This is a powerful debugging technique for understanding exactly what reaches the provider.

Quiz

Q1. What is the primary purpose of OpenRouter Broadcast?

Show answer

b — Broadcast pushes observability data to your platform automatically without requiring code changes.

Q2. Which of the following is NOT a supported Broadcast platform?

Show answer

Q3. What does debug: { echo_upstream_body: true } do?

Show answer

b — It echoes the transformed request body that OpenRouter sends to the upstream provider for debugging purposes.

Q4. What information does router metadata provide?

Show answer

b — Router metadata reveals which provider was chosen, what fallbacks were attempted, and which pipeline stages ran.

Q5. What data is included in a Broadcast trace?

Show answer

c — Traces include a comprehensive set of telemetry: latency, tokens, cost, model, provider, and error data.

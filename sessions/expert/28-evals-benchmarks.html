Session 28: Evals & Benchmarks

⏱ 45 min · 🚀 Expert · 📝 5 Quiz Questions
📖 Learn 🔬 Lab 📝 Quiz

Learn

Why Evals Matter

You cannot optimize what you cannot measure. Evals are how you know if a cheaper model is "good enough" for a task. Without evals, you're guessing — with evals, you're engineering.

Eval-Driven Optimization

The process is simple and iterative: define quality criteria → run eval → try different model → compare scores. Each cycle moves you closer to the optimal balance of cost, latency, and quality for your specific use case.

What to Measure

Key dimensions include: accuracy (correctness of output), faithfulness (stays on topic, doesn't hallucinate), format compliance (follows the expected schema), latency (time to first token), and cost (token usage per task). Different tasks prioritize different metrics.

Prompt Evaluation

Compare system prompt variants systematically. Keep the model fixed, change only the prompt, and measure which version produces the best scores. This removes model variability from prompt engineering decisions.

Model Comparison

Run the same prompt through multiple models, score results against ground truth. This gives you an apples-to-apples comparison of model capability for your specific task — not generic benchmark scores, but task-specific performance data.

The Eval Loop

The standard evaluation loop has five steps: 1) Create eval dataset (inputs + expected outputs). 2) Run pipeline against it. 3) Score results. 4) Adjust model/prompt. 5) Repeat. This turns model selection from a subjective choice into an empirical process.

Design Arena Benchmarks

OpenRouter provides Design Arena rankings (ELO, win rate) for some models. These community-driven benchmarks give a signal about relative model quality, but should be supplemented with your own task-specific evals for production decisions.

Lab

Build a simple eval harness that compares multiple models across several task categories:

python3 -c "
import requests, os, json, time

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

eval_cases = [
    {'input': 'What is 2+2?', 'expected_contains': '4', 'category': 'math'},
    {'input': 'What is the capital of France?', 'expected_contains': 'Paris', 'category': 'geography'},
    {'input': 'What color is the sky on a clear day?', 'expected_contains': 'blue', 'category': 'science'},
]

models_to_eval = ['openai/gpt-4o-mini:free', 'google/gemini-2.0-flash-exp:free']

def eval_model(model_id):
    results = []
    for case in eval_cases:
        start = time.time()
        resp = requests.post(url, headers=headers, json={
            'model': model_id,
            'messages': [{'role': 'user', 'content': case['input']}],
            'max_tokens': 100,
        })
        elapsed = time.time() - start
        content = resp.json()['choices'][0]['message']['content']
        tokens = resp.json()['usage']['total_tokens']
        passed = case['expected_contains'].lower() in content.lower()
        results.append({'category': case['category'], 'passed': passed, 'time': elapsed, 'tokens': tokens})

    score = sum(r['passed'] for r in results) / len(results)
    avg_time = sum(r['time'] for r in results) / len(results)
    print(f'\n{model_id}:')
    print(f'  Score: {score*100:.0f}% ({sum(r[\"passed\"] for r in results)}/{len(results)})')
    print(f'  Avg time: {avg_time:.2f}s')
    for r in results:
        print(f'  {r[\"category\"]}: {\"✅\" if r[\"passed\"] else \"❌\"} ({r[\"time\"]:.2f}s, {r[\"tokens\"]} tokens)')

for model in models_to_eval:
    eval_model(model)
"

Try this: Add more eval cases covering different categories, or add more models to compare. The scoring function can also be made more sophisticated — check for semantic similarity instead of substring matching using an embedding model.

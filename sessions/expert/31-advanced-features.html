Session 31: Advanced Features

⏱ 45 min · 🚀 Expert · 📝 5 Quiz Questions

Learn

Response Caching

Cache identical LLM requests to reduce cost and latency for repeated queries. This is especially useful for agent workflows, unit testing, and any scenario where the same input occurs multiple times. Cached responses return instantly and cost nothing.

Response Healing Plugin

Automatically validates and repairs malformed JSON responses. When a model produces invalid JSON (trailing commas, missing brackets, truncated output), the response healing plugin fixes it before returning. This makes structured outputs dramatically more reliable in production.

Custom Classifiers

Tag generations by department, task type, or complexity level. Custom classifiers let you attach metadata to every request for cost attribution, usage analytics, and audit trails. This is essential for organizations that need to track AI spend by team or project.

Embeddings

Generate vector embeddings from text and images. Access multiple embedding models through a single API — no need to manage separate endpoints for different embedding providers. Embeddings power semantic search, clustering, and RAG pipelines.

Responses API (Beta)

OpenAI-compatible Responses API with support for reasoning, tool calling, and web search. This gives you access to the latest OpenAI response generation capabilities through the OpenRouter endpoint, with the same provider flexibility you get with chat completions.

App Attribution

Use HTTP-Referer and X-OpenRouter-Title headers to identify your app in OpenRouter's analytics. This helps your app appear in the Design Arena rankings and gives you visibility into how your application is using the API.

Lab

Try response caching and response healing to see how they improve reliability and performance:

python3 -c "
import requests, os, json

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

# Response healing — ensures valid JSON even if model makes mistakes
prompt = 'Generate a user profile for a person named Alice. Return as JSON with name, age, and email fields.'
resp = requests.post(url, headers=headers, json={
    'model': 'openai/gpt-4o-mini:free',
    'messages': [{'role': 'user', 'content': prompt}],
    'response_format': {'type': 'json_object'},
    'plugins': [{'id': 'response-healing'}],
})
data = resp.json()
content = data['choices'][0]['message']['content']
parsed = json.loads(content)
print('Healed JSON response:')
print(json.dumps(parsed, indent=2))

# App attribution
headers2 = {
    **headers,
    'HTTP-Referer': 'https://openroutermastery.dev',
    'X-OpenRouter-Title': 'OpenRouter Mastery',
}
resp2 = requests.post(url, headers=headers2, json={
    'model': 'openai/gpt-4o-mini:free',
    'messages': [{'role': 'user', 'content': 'Say hello with attribution'}],
})
print('\nWith app attribution headers set ✓')
"

Try this: Remove the response_format parameter and see if the model still returns valid JSON. Then add response healing back and notice how it repairs common JSON issues like trailing commas or missing closing braces.

Session 26: Workspaces, Budgets & Presets

⏱ 35 min · 🚀 Expert · 📝 5 Quiz Questions

Learn

Workspaces

Workspaces are separate environments with their own API keys, routing defaults, guardrails, and observability settings. They let you isolate different environments without managing multiple accounts. A typical setup:

  • Dev workspace — No budget limits, full access to all models, minimal guardrails
  • Staging workspace — Limited budget, all requests logged, standard guardrails
  • Production workspace — Hard budget caps, strict guardrails enabled, monitoring configured

Budgets

Set daily, weekly, monthly, or lifetime caps on spending. Budgets are evaluated at the workspace level and can be hard (block requests when met) or soft (send alerts). You can set different budgets for different keys within the same workspace. This is essential for cost governance in teams.

Presets

Presets let you save model configurations — including the model selection, system prompt, temperature, max tokens, response format, and provider preferences — once and reuse them across any application. Change the preset, and every app using it updates automatically. No more copy-pasting configuration across codebases.

Custom Classifiers

Tag every generation with custom metadata — department, task type, complexity level, or any custom dimension. Classifiers appear in your logs and analytics, enabling you to slice usage by any tag. This is invaluable for chargebacks, cost allocation, and usage analysis across teams.

API Key Scoping

Create API keys scoped to specific workspaces with granular permissions. A key can be read-only, limited to specific models, restricted to certain presets, or given full access. This enables the principle of least privilege — each integration gets exactly the access it needs and nothing more.

Lab

Use a preset in a request and add classifier metadata:

python3 -c "
import requests, os

url = 'https://openrouter.ai/api/v1/chat/completions'
headers = {'Authorization': f'Bearer {os.environ[\"OPENROUTER_API_KEY\"]}'}

# In real usage, you would reference a preset from the dashboard:
resp = requests.post(url, headers=headers, json={
    'model': 'openai/gpt-4o-mini:free',
    'messages': [{'role': 'user', 'content': 'Write a summary of this session.'}],
    # In production, use a saved preset:
    # 'preset': 'my-production-preset',
})

data = resp.json()
print(f'Model: {data[\"model\"]}')
print(f'Response: {data[\"choices\"][0][\"message\"][\"content\"][:200]}')

# Query available models for workspace-level decisions
models_resp = requests.get('https://openrouter.ai/api/v1/models?sort=pricing-low-to-high')
cheapest = models_resp.json()['data'][:3]
print('\nCheapest models (good for workspace defaults):')
for m in cheapest:
    print(f'  {m[\"id\"]} — context: {m.get(\"context_length\", \"?\")} tokens')
"

Try this: Visit the OpenRouter dashboard to create a preset with a specific system prompt and model configuration. Then modify the lab to reference your preset ID and see how the configuration applies automatically.
